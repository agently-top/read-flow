# Read Flow 管理员功能快速入门

## 5 分钟快速配置

### 步骤 1: 配置管理员密码

编辑 `.env` 文件，添加管理员密码：

```bash
cd /root/.openclaw/workspace/read-flow

# 编辑 .env 文件
echo "ADMIN_PASSWORD=MySecurePassword2026" >> .env
```

或者手动编辑：
```bash
nano .env
# 添加一行：ADMIN_PASSWORD=你的密码
```

### 步骤 2: 重启后端服务

```bash
# 如果使用 Docker
docker-compose restart backend

# 或者直接运行后端
cd backend
npm run dev
```

### 步骤 3: 访问管理员登录

打开浏览器访问：
```
http://localhost:3000/admin/login
```

输入刚才设置的密码，点击登录。

### 步骤 4: 开始审阅文章

登录后会自动跳转到管理员仪表板：
```
http://localhost:3000/admin/dashboard
```

**使用方法：**
1. 左侧显示所有待审阅文章
2. 点击任意文章，右侧显示内容
3. 选择审阅操作：
   - ✅ 精读 - 值得深入阅读的文章
   - ⛔ 跳过 - 不感兴趣或低质量
   - ⏰ 稍后 - 暂时跳过，稍后处理
4. 可选：添加审阅笔记
5. 文章自动标记并同步到思源笔记

## 常见问题

### Q: 忘记密码怎么办？

A: 直接修改 `.env` 文件中的 `ADMIN_PASSWORD`，然后重启后端服务。

### Q: 会话多久过期？

A: 24 小时。过期后需要重新登录。

### Q: 可以在多个设备登录吗？

A: 可以。每个设备会生成独立的 session token。

### Q: 如何退出登录？

A: 点击右上角的"退出登录"按钮。

### Q: 精读文章如何同步到思源笔记？

A: 确保配置了 `SIYUAN_NOTEBOOK_ID` 环境变量，精读操作会自动同步。

## 安全提示

⚠️ **重要安全建议：**

1. **使用强密码** - 至少 12 位，包含大小写字母、数字和符号
2. **生产环境使用 HTTPS** - 防止密码在传输中被窃取
3. **定期更换密码** - 建议每 3 个月更换一次
4. **不要分享密码** - 每个管理员应该有独立的访问方式
5. **检查访问日志** - 定期检查异常登录尝试
