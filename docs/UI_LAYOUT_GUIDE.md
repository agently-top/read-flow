
### 交互状态

```
初始状态:
- 密码框为空
- 登录按钮 enabled

输入中:
- 密码框显示输入内容（掩码）
- 登录按钮 enabled

- 登录按钮 disabled
- 显示 "登录中..."
- 按钮 opacity-50

成功:
- 跳转到 /admin/dashboard

失败:
- 显示错误提示（红色背景）
- 登录按钮恢复 enabled
- 密码框保持焦点
```

### 组件结构

```
AdminDashboard
├── Header
│   ├── Logo + Title
│   └── Navigation (Back link, Logout)
├── Sidebar (400px)
│   ├── Title "待审阅文章 (count)"
│   └── Article List
│       └── Article Items (clickable)
│           ├── Title (line-clamp-2)
│           └── Metadata
└── Main Content
    └── Review Panel (conditional)
        ├── Article Preview (if selected)
        │   ├── Title
        │   ├── Metadata
        │   ├── Summary
        │   ├── Content
        │   └── Original link
        └── Actions Card (sticky)
            ├── Action buttons (3)
            ├── Note textarea
            └── Info text
```

### 交互流程

```
1. 页面加载
   ↓
   - 验证 session token
   - 加载待审阅文章列表
   - 加载分类、标签、统计信息

2. 选择文章
   ↓
   - 点击左侧文章
   - 右侧显示文章预览
   - 文章高亮显示（border-gray-900）

3. 审阅操作
   ↓
   - 点击精读/跳过/稍后按钮
   - 发送 API 请求
   - 显示处理中状态
   - 成功后：
     * 从列表移除文章
     * 清空选择
     * 更新统计

4. 添加笔记
   ↓
   - 在 textarea 输入
   - 笔记随审阅操作一起提交

5. 退出登录
   ↓
   - 调用 logout API
   - 清除 localStorage
   - 跳转到首页
```

### 响应式断点

```
移动端 (< 768px):
- 单列布局
- 侧边栏在底部
- 全屏宽度

平板 (768px - 1024px):
- 单列布局
- 侧边栏在内容下方

桌面 (> 1024px):
- 双列布局
- 左侧内容 75%
- 右侧侧边栏 25%
- 侧边栏 sticky
```

## 6. 设计原则

### 一致性

- 使用统一的 Tailwind CSS 颜色方案
- 保持相同的圆角、阴影、间距
- 统一的字体大小和字重

### 可用性

- 清晰的视觉层次
- 明显的交互反馈（hover、active 状态）
- 加载状态和错误提示
- 响应式设计

### 可访问性

- 语义化 HTML 标签
- 足够的颜色对比度
- 键盘导航支持
- 屏幕阅读器友好

### 性能

- 静态生成（SSG）用于内容页面
- 动态渲染（SSR）用于个性化内容
- 按需加载数据
- 避免不必要的重渲染
